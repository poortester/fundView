# 私人基金工作台产品方向

## 核心目标

本项目不是自动交易系统，也不是普通聊天页。

核心目标是：围绕个人基金持仓，整合多数据源，形成可解释的持仓分析、风险预警、趋势判断和操作建议。

用户真正需要的是：
- 我现在的持仓结构是否合理。
- 今天行情对我的持仓有什么影响。
- 哪些基金需要重点观察。
- 是否存在主题重复、仓位过重、回撤扩大等风险。
- 未来可能怎么走，有哪些情景。
- 今天应该做什么，不应该做什么。
- AI 的建议事后看是否有效。

## 产品定位

一句话定位：

> 面向个人投资者的基金持仓分析与趋势研判工作台。

系统应优先服务于：
- 持仓分析
- 风险控制
- 仓位纪律
- 趋势情景预测
- 每日复盘
- 长期投资行为改进

系统不优先做：
- 自动下单
- 高频交易
- 股票短线追涨杀跌
- 无数据依据的 AI 闲聊
- 一句话式买卖指令

## 核心能力设计

### 1. 持仓画像

需要回答：
- 当前持有哪些基金。
- 每只基金的成本、净值、盈亏、仓位。
- 权益、债券、QDII、货币等资产比例。
- 行业/主题暴露情况。
- 是否存在同类基金重复持有。
- 单只基金或单一主题是否过重。

建议数据：
- 用户持仓
- 基金基础信息
- 基金分类
- 基金持仓行业
- 历史净值
- 估值涨跌

### 2. 多数据源行情分析

基金分析不能只看基金净值，需要多数据源综合判断。

第一阶段数据源：
- 基金净值/估值
- A 股主要指数：沪深300、中证500、创业板指、科创50
- 港股指数：恒生指数、恒生科技
- 美股指数：纳指、标普500
- 汇率：美元/人民币、港币/人民币
- 债券指标：十年期国债收益率、中债指数
- 商品避险指标：黄金、原油

第二阶段数据源：
- 基金经理变更
- 基金规模变化
- 基金持仓行业和重仓股
- 同类基金排名
- 新闻和公告
- 宏观数据
- 市场情绪指标

### 3. 趋势预测

这里的“预测”不能做成绝对涨跌判断，应做成情景预测和概率倾向。

建议输出形式：
- 短期趋势：未来 1-5 个交易日。
- 中期趋势：未来 1-4 周。
- 情景一：市场继续上涨，对持仓影响。
- 情景二：市场震荡，对持仓影响。
- 情景三：市场回撤，对持仓影响。
- 关键观察位：指数、估值、净值、仓位阈值。
- 置信度：低 / 中 / 高。
- 预测依据：数据源、历史走势、相关市场、仓位暴露。

不能输出：
- 保证收益。
- 确定性涨跌。
- 无依据的买卖指令。

### 4. 建议引擎

建议必须结构化，不只是自然语言。

每条建议应包含：
- 建议类型：补仓 / 持有 / 止盈 / 减仓 / 观察 / 暂停操作。
- 关联基金。
- 触发原因。
- 数据依据。
- 风险说明。
- 建议动作。
- 置信度。
- 有效期。
- 是否执行。
- 执行后结果。

建议触发规则示例：
- 单日估值跌幅超过阈值，但仓位未超限：可观察补仓。
- 单日上涨且浮盈较高：进入止盈观察。
- 同主题基金仓位过高：提示集中风险。
- 连续下跌但未到计划补仓区：提示不要提前加仓。
- 债券基金与权益基金相关性降低：提示防守仓有效。
- QDII 基金受汇率和海外指数双重影响：单独提示。

### 5. 每日复盘

每天应沉淀：
- 今日行情摘要。
- 今日持仓变化。
- AI 建议。
- 用户实际操作。
- 用户为什么操作或不操作。
- 明日观察点。
- 事后结果。

长期价值：
- 分析用户是否容易追涨。
- 分析用户是否过早补仓。
- 分析哪些建议有效。
- 找出真正影响收益的基金和行为。

## 推荐开发路线

### 阶段一：真实持仓和数据库接入

目标：
- 当前前端样例数据改为数据库读写。
- 支持基金、持仓、复盘、建议入库。

任务：
- 建立 API 层。
- 接入 Turso/libSQL。
- 持仓列表从 `holdings` / `funds` 读取。
- 新增基金写入数据库。
- 复盘写入 `review_notes`。
- 建议写入 `analysis_reports` 和 `advice_items`。

### 阶段二：多数据源采集

目标：
- 建立统一数据源接口。
- 先支持手动/模拟数据，再逐步接真实行情源。

任务：
- 设计 `data_providers` 抽象。
- 建立基金净值快照任务。
- 建立指数行情快照任务。
- 把原始数据保存到 `raw_payload`，便于追溯。

### 阶段三：建议引擎 2.0

目标：
- 从简单规则升级为“规则 + AI 解释”。

任务：
- 基础规则先判断是否触发。
- AI 只负责解释、归纳、补充风险，不直接越权下结论。
- 每条建议保存依据和置信度。

### 阶段四：趋势预测和情景推演

目标：
- 输出未来走势的多情景分析。

任务：
- 对基金历史净值、指数走势、相关市场做统计。
- 生成上涨/震荡/回撤三种情景。
- 估算每种情景下持仓收益和回撤。
- 生成关键观察位。

### 阶段五：个人投资记忆

目标：
- 让系统理解用户过去的操作习惯和偏差。

任务：
- 统计建议执行率。
- 统计执行后的收益和回撤。
- 识别追涨、杀跌、过早补仓、过度集中等行为。
- 生成个人投资纪律建议。

## 当前下一步

最应该做的是阶段一：

1. 增加后端 API。
2. 把基金和持仓从前端样例数据迁移到 Turso 数据库。
3. 让新增基金、持仓修改、复盘保存变成真实数据。
4. 保留现有建议引擎，但把建议结果入库。

完成阶段一后，再开始接多数据源和趋势预测，否则 AI 分析没有稳定的数据基础。
