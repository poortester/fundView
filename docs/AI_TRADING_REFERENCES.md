# AI 交易辅助参考项目记录

本文记录当前基金工作台可借鉴的外部 AI 交易/投研项目特性。用途是产品设计参考，不直接复制代码。

## 1. hsliuping/TradingAgents-CN

项目定位：
- 中文金融场景的多智能体交易/投研分析框架。
- 强调 A 股、港股、美股等中文用户常见市场。
- 适合借鉴多角色协作、研究报告结构、模型配置和中文金融表达。

可借鉴点：
- 多智能体分工：基本面分析、新闻分析、技术分析、风险分析、交易决策。
- 把 AI 输出组织成结构化报告，而不是只给一句买卖建议。
- 支持多 LLM 提供商和模型切换。
- 强调风险提示、研究用途和分析过程透明。

对本项目的启发：
- 当前基金工作台可以拆成多个分析角色：
  - 行情分析员：指数、行业、利率、汇率、外围市场。
  - 基金分析员：净值、估值、回撤、同类表现、基金经理。
  - 持仓风控员：仓位、主题重叠、单基金集中度、账户风险。
  - 操作建议员：补仓、持有、止盈、减仓、观察。
  - 复盘员：建议是否执行、执行效果、用户行为偏差。

注意事项：
- 该项目 README 明确存在混合许可证和商业授权限制，不能直接复制受限目录代码。
- 本项目应借鉴架构思想，不照搬实现。

## 2. oficcejo/aiagents-stock

项目定位：
- A 股股票盯盘和多 AI 分析系统。
- 更偏股票短线、板块轮动、龙虎榜、关键点位监控和报警。

可借鉴点：
- 多数据源盯盘。
- 批量分析多个标的。
- 关键点位监控和提醒。
- 板块/主题轮动分析。
- 生成较完整的股票分析和决策建议。

对本项目的启发：
- 基金系统不应照搬股票短线交易逻辑，但可以借鉴预警机制：
  - 基金估值大跌提醒。
  - 指数或主题板块突破/跌破关键位提醒。
  - 连续下跌或连续上涨提醒。
  - 持仓主题过度集中提醒。
  - QDII 基金结合汇率、海外指数、外围市场提醒。
  - 债券基金结合利率和债市环境提醒。

注意事项：
- 股票的龙虎榜、游资、短线点位不适合直接用于基金持仓。
- 本项目目标应是降低持仓决策噪音，而不是鼓励高频操作。

## 3. HKUDS/Vibe-Trading

项目定位：
- 把自然语言金融问题转成可运行分析的研究工作台。
- 覆盖数据加载、策略生成、回测、报告导出、持久化研究记忆。

可借鉴点：
- 自然语言提问驱动分析任务。
- 数据加载器与分析任务解耦。
- 策略/假设可以回测验证。
- 研究过程和结果可沉淀为长期记忆。
- 输出报告、运行卡片和审计记录。

对本项目的启发：
- 后续可支持自然语言问题：
  - 今天适不适合补仓我的白酒基金？
  - 我的消费基金是否重复持仓严重？
  - 最近 30 天建议是否有效？
  - 如果每跌 3% 补仓，过去一年结果如何？
  - 当前持仓未来一个月有哪些风险情景？
- 建议和预测必须可追溯到数据源和分析逻辑。
- 用户真实交易和复盘应沉淀为个人投资行为画像。

## 本项目取舍

应该借鉴：
- 多角色分析。
- 多数据源融合。
- 持仓风险预警。
- 自然语言研究入口。
- 报告沉淀和复盘记忆。
- 回测和情景推演。

不应该照搬：
- 自动交易。
- 股票短线指标主导的买卖建议。
- 只给结论、不解释依据的 AI 输出。
- 复杂重型多智能体框架的一次性全量实现。
- 未确认授权边界的外部代码。
